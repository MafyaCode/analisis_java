Menyelami Dunia Fungsi dalam Go

Sebuah perjalanan interaktif untuk memahami blok bangunan fundamental dalam bahasa pemrograman Go, dari sintaks dasar hingga konsep *first-class citizen* yang kuat.

Konsep Dasar

Deklarasi & Pemanggilan

Setiap fungsi di Go dideklarasikan dengan kata kunci `func`. Fungsi `main` adalah titik awal eksekusi program.

func sapa(nama string) { fmt.Println("Halo,", nama) } func main() { sapa("Dunia") }

Parameter: Value vs. Reference

Go meneruskan argumen secara *by value*. Untuk memodifikasi data asli, kita harus menggunakan *pointer* (mirip *by reference*).

Perubahan di dalam fungsi tidak mempengaruhi variabel asli.

func gantiNilai(val int) { val = 100 // Hanya mengubah salinan } x := 10 gantiNilai(x) // x masih bernilai 10

Perubahan di dalam fungsi akan mempengaruhi variabel asli.

func gantiNilaiPointer(ptr *int) { *ptr = 100 // Mengubah nilai asli } x := 10 gantiNilaiPointer(&x) // x sekarang bernilai 100

Nilai Kembali (Return)

Go unggul dengan kemampuannya mengembalikan beberapa nilai, pola yang umum untuk penanganan error.

func tambah(a, b int) int { return a + b }
func bagi(a, b float64) (float64, error) { if b == 0 { return 0, errors.New("pembagi nol") } return a / b, nil }

Konsep Lanjutan

Fungsi Variadic

Fungsi yang dapat menerima jumlah argumen tak terbatas untuk parameter terakhirnya. Di dalam fungsi, parameter ini menjadi sebuah *slice*.

func jumlahkan(angka ...int) int { total := 0 for _, n := range angka { total += n } return total } // Panggilan: jumlahkan(1, 2, 3)

Fungsi sebagai Nilai (*First-Class*)

Di Go, fungsi dapat diperlakukan seperti tipe data lain: disimpan dalam variabel, dilewatkan sebagai argumen, atau dikembalikan dari fungsi lain.

func main() { operasi := tambah // 'tambah' adalah func(int, int) int hasil := operasi(5, 3) // hasil = 8 }

Fungsi Anonim & *Closure*

Fungsi tanpa nama yang dapat didefinisikan secara *inline*. Ketika fungsi ini "mengingat" variabel dari lingkup luarnya, ia membentuk sebuah *closure*.

Ini sangat berguna untuk membuat "pabrik fungsi" (*function factory*).

func buatPengali(faktor int) func(int) int { return func(angka int) int { return angka * faktor // "mengingat" nilai 'faktor' } } func main() { kaliDua := buatPengali(2) fmt.Println(kaliDua(5)) // Output: 10 }

Membuat *counter* yang state-nya terjaga setiap kali dipanggil.

func buatCounter() func() int { i := 0 return func() int { i++ return i } } // counterA dan counterB punya state sendiri counterA := buatCounter() counterA() counterA()

Lapisan Konsep Fungsi Go

Memahami fungsi di Go adalah proses berlapis. Dimulai dari sintaks dasar, berlanjut ke fitur-fitur canggih yang memungkinkan pola pemrograman yang kuat dan fleksibel, hingga akhirnya menjadi dasar untuk konkurensi.

Langkah Selanjutnya

Setelah menguasai fungsi, jelajahi *method* dan *interface* untuk membuka kekuatan penuh pemrograman di Go, lalu selami *goroutine* dan *channel* untuk membangun aplikasi konkuren yang efisien.
